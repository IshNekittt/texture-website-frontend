# Pixel Textures - Frontend

Это клиентская часть веб-приложения "Pixel Textures", созданная с использованием React и Vite. Приложение представляет собой каталог цифровых текстур с возможностью фильтрации, сортировки и администрирования.

## Основные технологии

- **Framework**: React
- **Сборщик**: Vite
- **Управление состоянием**: Redux Toolkit
- **Маршрутизация**: React Router DOM
- **HTTP-клиент**: Axios
- **Стилизация**: CSS Modules (без CSS-фреймворков)
- **Анимации**: Framer Motion
- **Формы**: React Hook Form

## Начало работы

### 1. Клонирование репозитория

```bash
git clone https://github.com/IshNekittt/texture-website-frontend.git
cd pixel-textures-frontend
```

### 2. Установка зависимостей

Убедитесь, что у вас установлен Node.js (версия 18 или выше).

```bash
npm install
```

### 3. Настройка переменных окружения

Создайте файл `.env` в корневой папке проекта и добавьте в него переменную, указывающую на ваш бэкенд-сервер.

```dotenv
VITE_API_URL=http://localhost:3000
```

- Для локальной разработки используйте `http://localhost:3000`.
- Для продакшена укажите URL вашего развернутого бэкенд-сервиса (например, `https://some-url.com`).

### 4. Запуск проекта

Для запуска в режиме разработки выполните:

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173`.

## Скрипты

- `npm run dev`: Запускает сервер для разработки.
- `npm run build`: Собирает проект для продакшена.
- `npm run lint`: Запускает проверку кода с помощью ESLint.
- `npm run preview`: Запускает локальный сервер для предпросмотра продакшен-сборки.

## Структура проекта

- **`public/`**: Статические активы (иконки, изображения).
- **`src/`**: Основная папка с исходным кодом.
  - **`api/`**: Настройка клиента Axios для взаимодействия с бэкендом.
  - **`components/`**: Переиспользуемые React-компоненты (Header, Loader, TextureCard и т.д.).
  - **`contexts/`**: React Context для глобального состояния, не связанного с Redux.
  - **`pages/`**: Компоненты-страницы, соответствующие маршрутам (HomePage, CataloguePage, AdminLoginPage и т.д.).
  - **`redux/`**: Вся логика управления состоянием с помощью Redux Toolkit.
    - **`admin/`**, **`categories/`**, **`textures/`**, **`theme/`**: "Слайсы" (slices) состояния, разделенные по сущностям.
  - **`utils/`**: Вспомогательные функции (форматтеры, парсеры).
  - **`validation/`**: Схемы валидации (например, для форм).
